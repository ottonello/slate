    Http streaming endpoint

    GET /api/v1/listen HTTP/1.1
api-key: string

    The client can open a connection to this endpoint and start listening to incoming events using HTTP streaming. Notice this is NOT long polling, so when an event is received the connection is NOT immediately closed. The format of the events received will be the same as the ones used in the websocket connection.

    Websocket connection endpoint

    GET /api/v1/websocket HTTP/1.1
api-key: string
Content-Type: application/json

{
    "incoming message": {
        "type": "string",
        "event": {
            "from": "string",
            "body": "string"
        }
    },
    "message": {
        "type": "string",
        "event": {
            "to": [
                "string"
            ],
            "body": "string"
        }
    },
    "message result": {
        "type": "string",
        "event": {
            "id": "string"
        }
    },
    "error": {
        "type": "string",
        "event": {
            "code": "integer",
            "status": "string",
            "message": "string"
        }
    }
}

    The extension can use this endpoint to connect to a websocket to send and receive events. Authentication is performed during the initial handshake. If the api-key header is not provided, the handshake will be rejected.

    Parameters

    Name In Type Description
    api-key *  header string Extension API key.
    incoming message body WebSocketIncomingMessage Optional. An incoming message sent to the extension.
    message body WebSocketMessage Optional. An outgoing message sent by the extension.
    message result body WebSocketMessageResult Optional. Web Socket message sent result.
    error body WebSocketError Optional. Web Socket error result.
